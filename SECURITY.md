# Security Policy

## Supported versions
Security fixes are applied to the latest state of the default branch.

## Reporting a vulnerability
Please do not open public issues for suspected vulnerabilities.

Use one of these private channels:
1. GitHub Security Advisories (preferred): create a private vulnerability report in this repository.
2. Direct maintainer contact (if published in repository profile).

Include:
- A clear description of the issue.
- Impact assessment.
- Reproduction steps or proof of concept.
- Suggested remediation if known.

## Response expectations
- Initial triage target: within 5 business days.
- Confirmed vulnerabilities are fixed as quickly as possible and released with notes.
